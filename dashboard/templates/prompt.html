Global Prompts

Three prompts drive every crawl, and all are the same for every feature — only the placeholders differ. The distillation prompt summarizes a feature's docs into a brief and the keyword prompt derives scrape terms from them (both once per crawl); the base fit prompt scores each company using that brief.

Base fit prompt

Sent to Claude for every company assessment. Owns the scoring scale and output format. Per-feature nuance is added through each feature's Additional guidance, injected at {% raw %}{{FEATURE_INSTRUCTIONS}}{% endraw %}.

These placeholders are filled in automatically before each request. Keep the ones you want the model to see.
{% raw %}
{{FEATURE_NAME}}
The feature's name.
{{FEATURE_DOCUMENTATION}}
A compact brief distilled from the feature's documentation URL — not the full page — produced by the distillation prompt below. You can review or hand-edit each feature's brief in the Features editor.
{{FEATURE_INSTRUCTIONS}}
This feature's Additional guidance field (a short note is inserted when a feature has none).
{{CUSTOMER_DOMAIN}}
The company's root domain being assessed.
{{CRAWLED_CONTENT}}
The combined visible text of every page crawled on that one company's site, each section headed by its URL — not a single page. Gathered from up to 50 pages and capped at ~60,000 characters total. One fit assessment is produced per company from this whole bundle (not one per page).
{% endraw %}

Documentation distillation prompt

Run once per crawl to compress a feature's fetched documentation into the compact brief that feeds {% raw %}{{FEATURE_DOCUMENTATION}}{% endraw %} above. It's identical for every feature — only the documentation text differs. Editing it re-distills each feature's brief on its next crawl.

Keep {% raw %}{{DOCUMENTATION}}{% endraw %} in the text — without it the model never sees the docs.
{% raw %}
{{FEATURE_NAME}}
The feature's name.
{{DOCUMENTATION}}
The visible text fetched from the feature's documentation URL (the full page) — the raw input this prompt summarizes.
{% endraw %}

Keyword prompt

Run once per crawl to derive the list of terms the crawler counts on each prospect site (shown as the heatmap on the dashboard). Ask for a single-column CSV of keywords; the output is parsed one keyword per line. Same for every feature — only the documentation differs. Editing it regenerates each feature's keywords on its next crawl.

Keep {% raw %}{{DOCUMENTATION}}{% endraw %} in the text — without it the model has nothing to derive keywords from.
{% raw %}
{{FEATURE_NAME}}
The feature's name.
{{DOCUMENTATION}}
The visible text fetched from the feature's documentation URL (the full page).
{% endraw %}
