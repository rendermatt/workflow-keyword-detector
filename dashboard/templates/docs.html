What this app crawls

A quick reference for how pages are fetched and scanned for feature keywords.

How the crawl works

  1. You start a crawl from the dashboard's New Crawl button: pick a feature and upload a CSV of seed domains. A bare domain like example.com is treated as https://example.com.
  2. The workflow loads that feature's keywords from the database and, starting from each seed, fetches the page and follows links to other pages on the same domain — breadth-first — to reach documentation, blog posts, job postings, and the like. (15-second request timeout, bot User-Agent.)
  3. "Same domain" means the same registrable domain, including subdomains: from example.com it will crawl docs.example.com, example.com/careers, and so on. Off-domain links are ignored.
  4. On each page, <script> and <style> tags are stripped and the visible text is lowercased. For every keyword in the selected feature, it counts case-insensitive substring matches.
  5. Keyword counts are upserted per (feature, URL, keyword) and every fetched page is recorded, so re-crawling a feature refreshes the numbers instead of double-counting.
Bounded & polite. Each domain crawl stops after CRAWL_MAX_PAGES pages (default 50) and honors each host's robots.txt — disallowed URLs are skipped and logged on the domain's detail view. The crawler does not execute JavaScript, so content rendered only client-side won't be seen.
